说明

整理自 GNU LD MANUAL （ Version 2，Jan. 1994 ）

概述

GNU ld 将多个 object 和 archive files 链接起来，重新定位其中的 data，并将 symbol 引用一一对应。=ld= 有自己的 Linker Command Language 来配置链接的细节。

ld 通过 BFD 库来兼容不同的对象文件格式（如 COFF 或 elf 等）

命令行

有很多参数用来配置 ld 的行为，非参数外的内容被作为要链接的 object 文件。ld 会自动推测文件的格式类型。

常用命名行参数：

-o
指定生成文件名，如未指定，生成 a.out
-larchive
添加 archive 文件到要链接列表中，可多次引用
-b / --format
指定输入文件类型格式
-m
emulate the emulation linker，有默认值
-v / -V
显示版本， -V 同时显示支持的 emulations
-T
指定 linker sciprt

环境变量

有两个环境变量 GNUTARGET 和 LDEMULATION 对 ld 有影响

GNUTARGET
同 -b 参数，指定输入文件格式类型；设置为 default 将自动推测
LDEMULATION
同 -m ，使用 -v 查看有哪些可用的模拟选项

Command Language

显式控制链接过程，如

  1. input files
  2. file formats
  3. output file layout
  4. addresses of sections
  5. placement of common blocks

控制文件脚本被称为 linker script，通过 -T 参数指定。

linker scripts

由 statement 构成，基本操作有两种：一设置参数，二选择和分组

最常用的 statement 是 SECTIONS 命令，指定了输出文件的分布情况，是必选的。

MEMORY 命令补全 SECTIONS 功能，描述了目标架构中的一种可用内存分布

Expressions

基本的数学运算，类似于 C 的语法，不同之处在于

  • 只有整数，分为 long 和 unsigned long
  • 所有的常量是整数
  • 所有的 C 算法都支持
  • 可以引用、定义和创建全局变量
  • 可以调用通过目的内置函数

整数

可以后缀以 K 和 M 来表示相应大小

Symbol Names

正常的命名应该以字母、下划线或 point 开头，跟着任务长度的字母、数字、point 及横线。且不应该与关键字重名。

用双引号包起来的字符串可以不受上面条件的限制

Location Counter

特殊变量 '.' 表示当前位置计数器，表示输出 section 中的位置。只能用在 SECTION 命令中。该变量可以增加以生成空洞，但不能减少。

Evaluation

对表达式进行惰性求值。一般如变量开始地址，区域的内存大小会在加载时被首先计算。其它值则在存储位置确定后、被使用时计算。

赋值

symbol = expression ;
symbol &= expression ;
symbol += expression ;
symbol -= expression ;
symbol *= expression ;
symbol /= expression ;

末尾的分号是必须的。赋值操作可以出现在：

  1. ld script 中作为 command
  2. 在 SECTIONS 命令中作为独立的 statement
  3. 在 SECTIONS 中作为 section 定义的一部分，定义相对于 section 的地址

变量中保存的是绝对地址或可重定位地址：

绝对表达类型
symbol 中包括了的值，在整个输出文件中的偏移量
可重定位表达类型
相对于 section 的固定偏移量

表达式类型由其在脚本中所处的位置决定。在 section 定义中的赋值则为相对地址，在其它位置赋值则作为绝对符号。使用 ABSOLUTE 来显式地指定绝对符号。
