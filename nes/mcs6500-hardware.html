chapter 1. Microcomputer System

近来几年（1976年前后）大规模集成单板芯片发展迅速。其好处是能大大减少设计其个逻辑电路的成本，而只需要使用通用的硬件环境，通过 ROM 中的“程序”来适应不同的应用。

设计程序与设计逻辑电路殊途同归，分以下五个步骤

  1. 确定需求
  2. 设计结构，权衡其成本、性能及扩展性
  3. 完善细节
  4. 构造和验证原型
  5. 完成设计并指生产

基本构成

主要关注构成的基本元素，以及之间的数据关联。

对于计算机程序，一个主要的原则是分治。将复杂的任务拆分成足够简单的任务，通过一个周期内的指令来完成。

处理器

功能是取址、译址和执行指令。它通过

  1. 地址总线
  2. 双向的数据总线
  3. Write-Enable Switch

来控制整个系统及外设。此外 interrupt 机制使它能更好的完成异步操作，而不需要效率低下的轮循。

地址

地址表示处理器可以访问的位置。地址空间有 16 位，可访问 64k 空间。

将地址分为高低两个字节 ADH 和 ADL，即分为 256 个页，每个页有 256 个字节。

clock generator

用于提供心跳（拍子），使处理器和其它组件协同工作

program memory

ROM，存储指令，只读内存

data memory

RAM，易失。通过 Write-Enable 开关来控制读写

I/O 设备

与 RAM 类似，处理器通过地址进行读写操作

产品线

  1. MCS6501，兼容 M6800，16 位数据总线，8 位数据总线，有两个中断
  2. MCS6502, 同 6501，片上带时钟电路和驱动
  3. MCS605[3-5]，28 脚封装，其数据总线有所减少

核心概念

总线结构

总线分为地址和数据两种，

  1. 地址总线只能由操作器操作（单向写）
  2. 数据总线为双向读写，方向由 READ/WRITE LINE 控制

地址总线设定地址，数据总线从指定地址进行数据读写。两者通过双方波叠加时钟信号进行协调。

时钟分为两个相位：

  1. 相位一，修改地址总线信息
  2. 相位二，数据传输

中断

中断应该用在必要的地方，使外设能向处理器提出服务请求。中断分为 IRQ 和 NMI 两种，两者响应方式不同，但服务方式没有区别。

使用中断时，我们需要关注两个方面：

  1. 哪些信号需要中断
  2. 中断时需要触发何种服务
  3. 中断优先级
  • 什么时候需要中断

    需要中断的场景比如：

    1. Fully-Decoded Keyboard，其特征是不频繁，且处理简单
    2. Scanned Display，特征是周期性事务

    总结中断的应该场景多为：并发需要（如周期性事务）或不频繁的事件处理

  • 中断触发服务

    中断信号低电平时触发中断服务，在指令周期结束后被检查是否有中断信号。

    中断时处理器将保存上下文（如 PC 和 PSR 到栈），跳转到中断服务处执行指令

  • 中断优先级

    不同的中断源需要按优先级进行排列，需要有硬件的支持，支持软件的控制。有两种方式：

    1. 优先级编码器（硬件）：选中最高优先级的中断并排队
    2. polled interrupt：中断将触发 polling 中断，由软件决定进行何种操作

    另外硬件上可以通过 WIRE-OR 机制（类似于 x86 中断控制器）来同时接入多种中断。

IRQ

通过 PSR 寄存器中的 interrupt disable flag (I) 标记，标记在 RET 时由程序主动清除

NMI

在下降沿触发，不可屏蔽

RESET

RESET 信号应同时连接到处理器和外设的 RESET 口。

处理器重置后 PC 被重置，应首先初始化中断服务（其间不接收中断请求）。

另外需要保证外设的初始化状态正确。

处理器核心

处理器中重点关注 register 和 data paths。按照功能和地位的不同，分为两部分

  1. register section，具体的取址执行流程
  2. control section，其它操作
