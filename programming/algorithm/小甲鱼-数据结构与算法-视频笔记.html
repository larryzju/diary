线性表

线性表是一个有限的有序数列，每一个元素都有唯一有前驱和唯一的后继（除开头和结尾）

元素具有相同的数据类型。

抽象数据类型 ADT

相当于 class，指数学模型及定义在其上的一系列方法。抽象相对于具体，只指定其接口，而没有实现。

通过 ADT 提供的基本方法可以组合出更加复杂的操作

为什么有类型

  1. 适应不同的应用场景
  2. 节省空间（有限精度，如 int 和 long ）

类型分类

  1. 原子类型
  2. 组合类型

常用操作

  • init
  • clean
  • isEmpty
  • get(i)
  • locate(v)
  • insert(v)
  • delete(v)
  • length

例子

  • union(L1,L2)

实现方式

有两种存储结构来实现线性表

  1. 顺序表，使用连贯的内存地址来存放数据，需要记录数据的长度和空间的容量
  2. 链式表，不需要连续地址存放数据，用一个指针域指向下一个后继元素

链式表

包括两个基本域：数据域和指针域

习惯上留置一个头结点作为列表开头，可以在其数据域中存储列表的长度，这样有助于统一列表的访问接口。通过头结点可以访问整个链表，尾指针指向空（null）

单链表的建立方式

  1. 头插法，新建节点指向头节点，将链表指向新建节点，生成链表元素顺序与插入顺序相反
  2. 尾插法，维护一个临时的指向链表最后一个元素的 node，每次向 node.next 加入新元素，并将 node 后移

对单链表循环删除整表操作时，用一个临时变量来删除 

for p != nil {
    q := p.next
    free(p) // go 不需要显式的释放空间， p.next = nil
    p = q
}

单链实现代码如下所示（go 语言） 

package fishc

import "errors"

type Element struct {
	value interface{}
	next  *Element
}

type LinkedList struct {
	root Element
	len  int
}

func NewLinkedList() *LinkedList {
	return &LinkedList{}
}

func (s *LinkedList) IsEmpty() bool {
	return s.root.next == nil
}

func (s *LinkedList) Len() int {
	p := s.root.next
	len := 0
	for p != nil {
		p = p.next
		len++
	}
	return len
}

func (s *LinkedList) Insert(i int, v interface{}) (err error) {
	p := &s.root

	for p != nil && i > 0 {
		p = p.next
		i--
	}

	if p == nil {
		return errors.New("out of index")
	}

	p.next = &Element{v, p.next}
	return
}

func (s *LinkedList) Delete(v interface{}) (err error) {
	p := s.root.next
	for p.next != nil && p.next.value == v {
		p = p.next
	}

	p.next = p.next.next
	return
}

func (s *LinkedList) Get(i int) interface{} {
	p := s.root.next
	for j := 1; j < i; j++ {
		if p == nil {
			return nil
		}
		p = p.next
	}

	return p.value
}

func (s *LinkedList) ToArray() []interface{} {
	p := s.root.next
	len := s.Len()
	r := make([]interface{}, len)

	for i := 0; i < len; i++ {
		r[i] = p.value
		p = p.next
	}

	return r
}

单链表与顺序存储比较

比较项 单链表 顺序表
查找 O(n) O(1)
插入 O(1) O(n)
空间性能   预分配空间，造成空间浪费或溢出
  • 频繁查找，考虑用顺序存储
  • 频繁修改，考虑用链表存储
  • 元素个数不确定，或变化大，考虑用链表存储

静态链表

C 语言的指针提供了极大的灵活性（面向对象使用对象引用机制间接的实现了指针的某些功能），是实现链表的语言基础。

在早期（如 basic, fortran）中，以数组代替指针来描述单链表。这种方式称为游标实现法

  • 每个元素由游标和数据组成
  • 整体上 N 个元素构成，
  • 第 N-1 个元素的游标指向第一个元素
  • 第 0 个元素的游标指向最后一个元素
  • 第一个和最后一个元素 data 不存放数据
  • 其它元素的游标指向下一个元素的下标

代码实现如下所示 

package fishc

import "errors"

type StaticListElement struct {
	next  int
	value interface{}
}

type StaticList struct {
	elements []StaticListElement
	heap     int
	len      int
}

func NewStaticList(size int) *StaticList {
	s := &StaticList{}
	s.elements = make([]StaticListElement, size)

	// 初始化 elements
	for i := 1; i < size; i++ {
		s.elements[i].next = i + 1
	}

	// 初始时，头指针指向空
	s.elements[0].next = -1

	// heapHead 指向可用空间链表的头
	// 预留一个空的头，以统一操作方法
	// elements[0] 为空的列表头
	s.heap = 1

	// 最后一个元素指向 -1，表示 tail?
	s.elements[size-1].next = -1

	return s
}

func (s *StaticList) Get(i int) interface{} {
	h := 0

	for j := 0; j < i && h != -1; j++ {
		h = s.elements[h].next
	}

	if h == -1 {
		return nil
	} else {
		return s.elements[h].value
	}
}

func (s *StaticList) Len() int {
	return s.len
}

func (s *StaticList) ToArray() []interface{} {
	res := make([]interface{}, 0)
	h := s.elements[0].next

	for h != -1 {
		e := s.elements[h]
		res = append(res, e.value)
		h = e.next
	}
	return res
}

func (s *StaticList) alloc() (idx int, err error) {
	if s.heap == -1 {
		err = errors.New("no avaiable slot")
		return
	}

	idx = s.heap
	s.heap = s.elements[idx].next
	return
}

func (s *StaticList) free(idx int) {
	s.elements[idx].next = s.heap
	s.heap = idx
}

func (s *StaticList) Insert(i int, v interface{}) (err error) {
	h := 0

	for j := 0; j < i && h != -1; j++ {
		h = s.elements[h].next
	}

	if h == -1 {
		err = errors.New("index out of range")
		return
	}

	// 从可用的空 slots 中取出
	idx, err := s.alloc()
	if err != nil {
		return err
	}

	e := &s.elements[idx]
	e.value = v
	e.next = s.elements[h].next

	s.elements[h].next = idx
	s.len++
	return
}

func (s *StaticList) Delete(v interface{}) error {
	h := 0

	for h != -1 {
		next := s.elements[h].next
		if s.elements[next].value == v {
			break
		}
		h = next
	}

	if h == -1 {
		return errors.New("can't find value")
	}

	next := s.elements[h].next
	nextNext := s.elements[next].next
	s.elements[h].next = nextNext
	s.free(next)
	s.len--
	return nil
}

需要维护两个链表：

  1. 实际链表
  2. 备用链表

列表的操作：

  • 进行插入时，从备用链表中取出一个可用空间，插入到实际链表中
  • 进行删除时，删除元素放回到备用链表中
