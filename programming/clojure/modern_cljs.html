写作的原因

最近打算好好学习前端知识，奈何隔行如隔山，当年掌握的 jquery 知识在面对当前各种 react/vue/angularjs 等库和框架，无不令人眼花缭乱。

而 clojure 又是我一年以来下功夫使用和学习的语言，对其 lisp 语法和 REPL 开发方式特别满义。

虽然不很了解前端技术更新的路线，但就目前学习到的来看，主要有以下几个方面，clojurescript 似乎是个不错的选择

  1. javascript 有多种标准，且本身语法过于简单，而不够严格。实际上会以 typescript, coffeescript, es6 等写作，由编译器翻译成实际的 javascript 代码在浏览器中执行。
  2. virtual dom：DOM 标准过于具象化，直接在上面操作比较困难。一般在其上面抽象出一个虚拟的视图，程序操作虚拟视图，并由框架来映射到具体的 DOM 上。类似于 clojure 之与 JVM，C 之与处理器， luajit 之与 lua 语言？
  3. 统一 html, javascript, css 为统一的软件栈。如果后台是　clojure，那么甚至从前向后都可能统一起来。

之前学习 clojure 的过程中，借助它与 Java 互操作的特性，学习到了很多 java 和 jvm 的知识。特别是因为所在的角度不同，可能也因为资料相对较少，学习到的知识比较丰富，且充分。希望学习 clojurescript 也能帮我理解前端的技术发展到底痛点在哪里？解释为什么会有这么多的框架和技术演进。

NREPL

希望

clojure 最大的特点，从工程角度来讲，我认为是 NREPL 的渐进式开发流程。特别是像开发 java 程序时避免了大量的重复打包、启动、准备现场的流程。

学习 clojurescript 的一个主要的希望也是能够渐进式的，可以探索的方式进行开发。最好能够如同 lisp 开发一样，在编译器级别进行语言调试。

Tutorial 1

boot

modern cljs 中使用的是 boot 工具。之前比较熟悉 leiningen 工具，两者之前应该大部分是相通的。

目录结构

boot 的目录结构同 leiningen 相似

  1. built.boot
  2. src/cljs/<namespace>/core.cljs 代码目录
  3. html/index.html，类比于 resources 目录

注意 clojurescript 的后缀是 cljs

工具链

  • mac 上使用 brew install boot-clj
  • 或下载 boot.sh，第一次执行脚本时会从网络上获取，同 leiningen
  • 可恶的网络，几乎无法下载相关的包，只能配置 ss 服务来下载

工程文件

built.boot 是工程文件，类似于 leiningen 的 project.clj，sbt 的 built.sbt ，maven 的 pom.xml 

(set-env!
 ;; 指定源文件和资源文件路径
 :source-paths #{"src/cljs"}
 :resource-paths #{"html"}

 ;; 添加 plugin
 :dependencies '[[adzerk/boot-cljs "1.7.228-2"]])

;; 起别名，可以作为 boot 子命令，类似于 hdfs dfs 之类
(require '[adzerk/boot-cljs :refer [cljs]])

cljs 插件

编译 clojurescript 代码，并打包生成 main.js 文件，可以直接在 html 页面中 script tag 中加入。

使用命令 boot cljs target 来编译并生成到 target 目录中。其中包括

  1. 原始的静态资源和源码
  2. 一堆的 js 代码

boot 版本控制

命令 boot -V > boot.properties 生成一个控制文件，内容如下 

#http://boot-clj.com
#Thu Mar 09 14:56:52 CET 2017
BOOT_CLOJURE_NAME=org.clojure/clojure
BOOT_CLOJURE_VERSION=1.7.0
BOOT_VERSION=2.7.1

浏览结果

打开 target/index.html 浏览结果

boot 编译如同 org publish 一样，不会修改原来的 resources 和 src，只会在 target 目录下生成新的内容

boot-http
基于 http server 的 clojure 环境
boot-reload
自动重载静态文件
boot-cljs-repl
cljs 开发的 nrepl 环境
