Design

Basic

  1. Eliminate Data Redundancy
  2. Ensure Data Integrity and Accuracy

Primary Key

Can be a simple key or a composite key which should be

  1. Integer if possible
  2. AutoNumber
  3. not change
  4. unique
  5. not null

Relationship

  1. one-to-many: foreign key
  2. many-to-many: junction table
  3. one-to-one
