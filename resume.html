Project you want to talk?

I'd like to talk about the project named clj-gdata.

I am the leader of this project. I choose clojure as the major language, and use Spark as the compute engineer, use Parquet as the exchange persist storage file type.

The project provide three type of service

  1. nrepl port: which is TCP port used by clojure interrupt, so we can use editor to connect to remote cluster (spark driver), and submit our code snippet to interrupt. That is absolute wonderful than I expect so that we can dynamic change the runtime without restart or rebuild the large project.
  2. http port: as RESTFul service for frontpage usage. And also I made a simple and sometime ugly page for maintainer. I used Bootstrap, jQuery and AngularJS as the library. The service code is also written in clojure. I had considered to refactor it with ClojureScript (which is a compiler that translate clojure code to javascript, so we can write service and front in the same language).
  3. spark port: which is the redirect of the Spark management page. We can examine the process of current jobs and stages.

The main concept is Task/Model

Task is the abstract of the execution of Model.

Model is the interface that each developer of analysor or program must be comply. The core method of the Model is the description (metadata), run method, check method, and dynamic ui form generator.

The Task is worked in seperated thread in driver and will be submit to Spark's job and run in seperated. The result of task is saved as Parquet file in HDFS, and will record in memory HashTable and also persist metadata in HDFS.

Task result can be serailized by use general model such as filter, join, diff and so on.

In my desgin, I want to make a high level abstract of Topology, which can be describe by DAG and parameters, and can be seen as a blackbox model. But until now, I don't have the opptourtiny to implement my desgin.

This project involved so many component hadoop ecosystem. And thanks for clojure's NREPL feature and JVM based language. I explore the API of HBase, JDBC, Elasticsearch, Zookeeper, HDFS, Spark carefully. And learned and read source code of part of there project.

And also I am fans of Lisp, but I don't have oppournity to use it in real project. But I can proudly say clojure is the best choice for this application.

How do you evalute SAP?

SQL 主键 外键

  • 主键唯一性约束，只能有一个
  • 外键是指向另一个表的主键，可以有重复，用来作 join，可以有多个
  • 索引 排序信息，提高查询速度，可以有多个

数据库索引优缺点

  • 提高检索速度
  • 可用于保证唯一性
  • 可用于保证一致性（连接）
  • 降低了插入、修改、删除速度
  • 维护索引需要时间

SQL 连接

  • 内连接
  • 左外连接
  • 右外连接
  • 全外连接
  • 左半连接

指针与引用

  • 引用必须被初始化，指针不必
  • 引用初始化以后不能被改变，指针可以改变
  • 不存在指向空值的引用，但是存在指向空值的指针
  • 按值传参和按引用传参，效率不同
  • 引用的创建和销毁不会调用类的拷贝构造函数和析构函数
