Quick Start

create a project

kubebuilder init --domain my.domain
# (optional)
# go mod init <modulename>

create an API

kubebuilder create api --group webapp --version v1 --kind Guestbook
# create api/v1/guestbook_types.go and controller/guestbook_controller.go

test it out locally

(optional) use kind to get a local cluster for teting

set current context

Use kubectl cluster-info to check

install CRD

make install

run your controller

make run

install samples

kubectl apply -f config/samples/

run it on the cluster

build and push

make docker-build docker-push IMG=<some-registry>/controller

deploy to cluster

make deploy

group & version

  • API Group is simply a collection of related functionality
  • Each group has one or more versions

kind and resources

  • Each API-group-version contains one or more API types, which we call Kinds
  • Resource is simply a use of a Kind in the API. Often there's a one-to-one mapping between Kinds and resources
  • Sometimes, the same Kind may be returned by multiple resources
  • With CRDs, however, each Kind will correspond to a single resource.
